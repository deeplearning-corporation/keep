# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|----------|
| 1.0.x | ✅ 积极支持 |
| < 1.0 | ❌ 不支持 |

## 报告漏洞

我们非常重视 KEEP 编译器的安全性。如果您发现安全漏洞，请通过以下方式报告：

### 不要公开披露

请不要在 GitHub issues 中公开报告安全漏洞。这可能会让其他用户面临风险。

### 报告方式

1. **发送 Microsoft**：YS_MFAYGMGRXYG@outlook
   - 简单描述安全问题

2. **报告 QQ**：QQ 347709188

### 报告内容

请包含：
- 漏洞类型和描述
- 受影响的版本
- 复现步骤
- 潜在影响
- 如果可用，提供概念验证代码

### 响应时间

- 24小时内：确认收到报告
- 7天内：提供初步评估
- 30天内：发布修复或缓解措施

## 漏洞披露政策

我们遵循协调披露原则：
1. 确认漏洞后，我们将与您合作了解影响
2. 我们将在修复后通知受影响的用户
3. 在修复发布后，我们将公开致谢（如果您同意）

## 安全更新

安全更新将通过：
- GitHub Releases 页面
- 邮件通知给注册用户
- 安全公告邮件列表
